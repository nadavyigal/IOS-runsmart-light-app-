# Live AI Coach Endpoint

Status: Implemented, deployed, and remotely smoke-tested
Date: 2026-05-18

## Summary

RunSmart Coach now has a repo-local Supabase Edge Function contract at `supabase/functions/coach_message/index.ts`.

The endpoint is authenticated, stores both user and assistant messages, calls OpenAI server-side when configured, and returns a deterministic fallback assistant message when OpenAI is unavailable. The iOS app still keeps its local deterministic fallback path for missing auth, missing endpoint, network failure, malformed response, or backend outage.

## Endpoint

- Path: `POST /functions/v1/coach_message`
- Auth: Supabase user JWT in `Authorization: Bearer <JWT>`
- Client headers: `apikey: <Supabase publishable key>`, `Content-Type: application/json`
- Secrets: `OPENAI_API_KEY`; optional `OPENAI_MODEL` defaults to `gpt-4.1-mini`

## Request DTO

```json
{
  "conversationId": "optional UUID",
  "clientMessageId": "UUID generated by iOS",
  "entryPoint": "today | plan | run | report | profile",
  "message": "Should I still run today?",
  "context": {
    "generatedAt": "2026-05-18T09:30:00Z",
    "entryPoint": "today",
    "runner": {},
    "today": {},
    "plan": {},
    "recovery": {},
    "wellness": {},
    "activity": {},
    "routes": [],
    "reports": [],
    "limitations": []
  },
  "clientTimestamp": "2026-05-18T09:30:00Z"
}
```

The iOS DTO intentionally sends training summaries only. It does not encode raw route coordinates, latitude, longitude, or route point arrays. The Edge Function also rejects requests containing coordinate-like keys such as `latitude`, `longitude`, `routePoints`, `coordinates`, `polyline`, `gps`, or raw `points`.

## Response DTO

```json
{
  "conversationId": "UUID",
  "userMessageId": "UUID",
  "assistantMessage": {
    "id": "UUID",
    "role": "assistant",
    "content": "Short Coach answer",
    "createdAt": "2026-05-18T09:30:01Z"
  },
  "source": "live_ai",
  "fallback": false,
  "suggestedAction": null,
  "safetyFlags": [],
  "usage": {
    "inputTokens": 1000,
    "outputTokens": 120,
    "totalTokens": 1120
  }
}
```

`source` is `live_ai` when OpenAI succeeds and `fallback` when the backend deterministic fallback responds.

## Persistence

- Live path: backend stores both the user message and assistant message.
- iOS fallback path: iOS stores the deterministic fallback turn only when the backend is unavailable or cannot return a valid response.
- Idempotency: iOS sends one `clientMessageId` per user turn. The backend stores the user row with that ID and the assistant row with `<clientMessageId>:assistant`.
- Backward compatibility: conversations still populate `profile_id = auth.uid().uuidString`; the migration also adds `auth_user_id`.
- History reads: iOS reads persisted Coach turns through `coach_messages_for_conversation`, an authenticated owner-filtered RPC. Anonymous execute is revoked; authenticated execute remains enabled.

## Deployment

```bash
export SUPABASE_ACCESS_TOKEN=...
npx -y supabase@latest link --project-ref dxqglotcyirxzyqaxqln
npx -y supabase@latest secrets set OPENAI_API_KEY="$OPENAI_API_KEY" OPENAI_MODEL="${OPENAI_MODEL:-gpt-4.1-mini}" --project-ref dxqglotcyirxzyqaxqln
npx -y supabase@latest functions serve coach_message --env-file local.env
npx -y supabase@latest functions deploy coach_message --project-ref dxqglotcyirxzyqaxqln --use-api --no-verify-jwt
```

The schema migrations and Edge Function have already been applied/deployed remotely on 2026-05-18. Run `db push` only when reconciling a fresh/local Supabase project.

The repo also includes `scripts/deploy-coach-message.sh`, which reads ignored `local.env`, verifies required variables, sets only the Coach OpenAI secrets, and deploys `coach_message`. Add `SUPABASE_ACCESS_TOKEN` to `local.env` or export it in the shell before running the script.

## Limitations

- This is live AI chat before/after runs, not live AI coaching during runs.
- The endpoint does not mutate training plans or apply suggested actions.
- Medical symptoms trigger conservative stop-and-consult guidance, not diagnosis or treatment.
- Edge Function local execution depends on the Supabase CLI and deployed database migration.
- Manual in-app QA is still required before beta sign-off: Today entry, Report entry, thread reload, backend-broken fallback, duplicate-row check, and stored metadata inspection for no raw coordinates.
