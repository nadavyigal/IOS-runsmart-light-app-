# Mobile UX Standards

- RunSmart iOS should feel native, premium, simple, fast, and visual.
- Make Today the daily command center.
- Make the next action obvious.
- Prefer progressive disclosure over dense dashboards.
- Use cards, summaries, and hierarchy to make scanning easy.
- Keep training plans understandable without coaching jargon.
- Make AI coaching specific, concise, and useful.
- Use native gestures where they improve the flow.
- Support small iPhones.
- Avoid generic fitness-app visuals and web-app clutter.
- Profile should help the runner manage goals, integrations, privacy, and account settings without becoming a dumping ground.
