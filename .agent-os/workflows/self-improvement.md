# Self-Improvement Workflow

Use after user corrections, failed implementations, broken builds/tests, bad assumptions, bad SwiftUI patterns, repeated mistakes, confusing output, overengineering, or missed requirements.

## Steps
1. Name what happened in one sentence.
2. Identify the broken assumption or process gap.
3. Add a short entry to `tasks/lessons.md`.
4. Convert it into a practical future rule.
5. Apply that rule to the current or next similar task.

## Lesson Format
Use `.agent-os/templates/lesson-template.md`.

## Done
- Lesson added.
- Future rule is specific enough to change agent behavior.
