# Feature Planning Workflow

Use for ideas, product changes, and non-trivial iOS features.

## Read
- `AGENTS.md`
- Agent router: `CLAUDE.md` or `CODEX.md`
- `tasks/lessons.md`
- `docs/product/current-product-state.md`
- `docs/architecture/current-ios-architecture.md`
- `.agent-os/templates/product-brief-template.md`
- `.agent-os/templates/feature-spec-template.md`

## Steps
1. Restate the idea and outcome.
2. Identify the runner problem and daily-use moment.
3. Confirm whether this should be native iOS, web parity, or intentionally different.
4. Draft a short product brief.
5. Draft a feature spec with scope, non-goals, states, data, permissions, and QA.
6. Break into small development stories.
7. Update `tasks/todo.md`.
8. Save approved specs under `docs/specs/`.

## Done
- Product brief exists.
- Feature spec exists or is ready for user approval.
- Stories are small enough to implement and verify one at a time.
