# PR Review Workflow

## Read
- `AGENTS.md`
- `tasks/todo.md`
- `tasks/session-log.md`
- `.agent-os/templates/pr-summary-template.md`
- Relevant git diff only

## Steps
1. Inspect changed files.
2. Summarize user-facing and technical changes.
3. List verification performed.
4. List risks, follow-ups, and known gaps.
5. Ensure no unrelated changes are included.

## Done
- PR summary is concise, accurate, and includes validation.
