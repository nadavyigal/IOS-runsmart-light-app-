# TestFlight Review Workflow

## Read
- `AGENTS.md`
- `tasks/lessons.md`
- `docs/qa/testflight-checklist.md`
- `docs/qa/app-store-readiness-checklist.md`
- `.agent-os/templates/testflight-report-template.md`

## Steps
1. Confirm authoritative project, scheme, app name, bundle id, team, version, and build number.
2. Check entitlements and usage descriptions against actual app features.
3. Verify privacy, HealthKit, location, payments, and sign-in implications.
4. Run Release archive or document why not run.
5. Run smoke test on the archived build when practical.
6. Produce TestFlight readiness report.
7. Update lessons for any release-blocking surprise.

## Done
- Report clearly says Ready, Not Ready, or Ready With Risks.
